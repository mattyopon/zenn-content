## TL;DR

React Native (Expo) + TypeScript で12本のモバイルアプリを開発しました。AI系からユーティリティまで、短期間で量産するためのテンプレート戦略について書きます。

## 開発したアプリ一覧

| アプリ | 概要 |
|-------|------|
| **AI Journal** | AI がジャーナリングをサポート |
| **AI Meal Planner** | AI 食事プランニング |
| **AI Study Buddy** | AI 学習サポート |
| **AI Wallpaper** | AI 壁紙生成 |
| **Focus Flow** | ポモドーロタイマー |
| **Pet Care Log** | ペットのお世話記録 |
| **QR Tool Pro** | QRコード生成・スキャン |
| **Sleep Sounds Pro** | 睡眠サウンド・瞑想 |
| **SubTracker** | サブスクリプション管理 |
| **Water Reminder** | 水分補給リマインダー |

## 量産戦略

### テンプレートプロジェクト
共通のテンプレートを作り、以下を標準化しました：

- **Expo Router** - ファイルベースルーティング
- **TypeScript** - 型安全性
- **共通コンポーネント** - ボタン、カード、モーダル
- **テーマシステム** - ダーク/ライトモード対応
- **ストア設定** - アイコン、スプラッシュ、メタデータ

### AI系アプリのパターン
4本のAIアプリは共通のアーキテクチャを使っています：

```
ユーザー入力 → APIリクエスト → AI応答 → UI表示 → ローカル保存
```

APIクライアント、レスポンスパーサー、ローカルストレージを共通化し、UIとプロンプトだけをアプリごとにカスタマイズ。

## 学び

- **Expo は最強** - ビルド、OTAアップデート、Push通知が全て統合
- **テンプレート化が鍵** - 1本目に時間をかけ、2本目以降は爆速
- **AI機能は差別化要因** - 同じカテゴリでもAI搭載で価値が変わる

## まとめ

モバイルアプリの量産は「いかに共通部分を再利用するか」が全てです。Expo + TypeScript のテンプレート戦略により、1アプリあたりの開発時間を大幅に短縮できました。
